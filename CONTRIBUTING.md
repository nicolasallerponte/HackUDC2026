# Guía de Contribución
 
¡Gracias por tu interés en contribuir a nuestro proyecto del **HackUDC 2026**! Somos Xoel, Miguel, Jacobo y Nicolás, y estamos encantados de que quieras formar parte de esto.
 
Como este es un proyecto de naturaleza Open Source nacido en un hackathon, damos la bienvenida a cualquier aportación: desde corregir un typo en la documentación hasta añadir nuevas funcionalidades o resolver bugs.
 
## ¿Cómo contribuir?
 
### 1. Reportar Bugs o Sugerir Mejoras
Si encuentras un error o tienes una idea genial, por favor abre un **Issue** en nuestro repositorio. 
* Usa un título descriptivo.
* Explica los pasos para reproducir el error (si es un bug).
* Describe claramente la mejora que sugieres.
 
### 2. Contribuir con Código (Pull Requests)
Si quieres mancharte las manos de código, ¡genial! Sigue estos pasos:
 
1. **Haz un Fork** del repositorio.
2. **Clona** el repositorio en tu máquina local.
3. **Crea una rama** para tu funcionalidad o corrección:
   `git checkout -b feature/nueva-funcionalidad` o `git checkout -b fix/correccion-error`
4. **Haz tus cambios** y asegúrate de que el código funciona correctamente.
5. **Haz commits** con mensajes claros y descriptivos:
   `git commit -m "feat: añade sistema de login"`.
6. **Sube tus cambios** a tu fork:
   `git push origin feature/nueva-funcionalidad`
7. **Abre un Pull Request (PR)** hacia la rama `main` de nuestro repositorio.
 
### Reglas de Estilo
* Intenta mantener el mismo estilo de código que ya existe en el proyecto.
* Comenta tu código cuando hagas operaciones complejas.
* Actualiza el `README.md` si tus cambios afectan a la instalación o uso del proyecto.
 
¿Tienes dudas? ¡Escríbenos a cualquiera de nuestros correos y te echaremos una mano!
